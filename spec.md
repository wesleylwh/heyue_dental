# 禾悦口腔平板导览 — 页面开发规格

> 这是面向平板讲解的口腔门诊展示工具，不是线上获客官网。
> 设计决策都应以"前台拿着平板给客户讲"为场景出发。

---

## 全局约束

- 主色：`emerald-500`（绿）/ `brand-primary`（蓝，历史遗留，种植价格部分保留）
- 圆角：卡片 `rounded-[32px]`，大区块 `rounded-[40px]`，小元素 `rounded-2xl`
- 版心：`max-w-6xl mx-auto`
- 按钮最小高度：`py-4`（≥ 44px），适合触控
- 动画：用 `motion/react`，`whileHover`, `whileInView` 轻量使用
- 字号规范：一级标题 `text-5xl/6xl`，二级 `text-3xl/4xl`，正文 `text-lg/xl`，标签 `text-sm`
- 价格：默认不展示（sanitizeProject 在 App.tsx 自动处理），种植板块例外

## 二、服务项目（Projects.tsx）— 重点重构

### 目标结构

```
[Header]
  标题：服务项目
  副标题：根据您的需求，选择适合的诊疗方向

[双入口 Tab]
  Tab A：按您的情况找
  Tab B：按诊疗方向找

[Tab A 内容：6 张问题卡片]
  - 缺牙了怎么办 → 种植 / 义齿
  - 牙齿不齐 → 正畸
  - 牙疼 · 蛀牙 → 补牙 · 根管
  - 牙龈出血 · 牙周问题 → 牙周治疗
  - 想洗牙 · 做检查 → 检查 · 洁牙
  - 牙齿缺损 · 想改善外观 → 修复 · 美学

  每张卡片：
  - 大号问题描述
  - 对应方向标签
  - 一句话说明
  - → 查看方案 按钮（滚动至对应 board）

[Tab B 内容：6 个服务板块卡片]
  - 补牙 · 根管
  - 拔牙 · 外科
  - 修复 · 美学
  - 牙周 · 洁牙
  - 矫正中心
  - 种植中心（特殊样式）

  每张卡片：
  - 板块名称
  - 适合情况（1-2 句）
  - 核心项目标签（3-4 个）
  - → 了解详情 按钮（跳转 ProjectDetail）

[种植品牌看板]（独立区块，保留现有内容）
```

### 数据来源
- Tab B 卡片复用 `CORE_SERVICE_BOARDS`（constants.ts 已有）
- 种植看板复用 `IMPLANT_PRICE_GUIDE`
- Tab A 问题卡片：硬编码在 Projects.tsx

### 移除内容
- "当前开发状态"占位卡（01/02/03 说明）
- "这些服务板块后面会逐块加入"提示区

## 四、诊所实力（About.tsx）— 中量改造

### 保留
- 品牌信息面板（卡片右侧深色区域）
- 诊所故事文段
- "我们坚持的方式" 3 项

### 修改
- 顶部 badge：`关于禾悦` → `诊所实力`
- 主标题调整：突出"实力"维度而非只有故事

### 新增区块

#### A. 就诊流程（5步横向步骤条）
```
01 初诊检查 → 02 方案沟通 → 03 治疗实施 → 04 复查跟进 → 05 术后回访
```
- 横向步骤条，平板横屏效果好
- 每步：数字编号 + 标题 + 一句话说明

#### B. 服务保障（4 项图标卡片）
- 透明报价：治疗前确认费用，无隐形收费
- 无菌操作：一人一机一消毒
- 治疗记录：完整病历留存，便于复诊
- 术后回访：治疗后主动跟进恢复情况

#### C. FAQ（4 个高频问题，手风琴/展开式）
1. 第一次来需要预约吗？
2. 初诊检查收费吗？
3. 种植/矫正大概要多久？
4. 是否支持分期付款？
