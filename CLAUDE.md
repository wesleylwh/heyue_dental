# 禾悦口腔 · 平板导览 H5 — 开发规范

## 产品定位

**这不是一个官网，是一个平板讲解工具。**

- 使用场景：前台 / 咨询师 / 医生拿着平板给来诊客户讲解
- 辅助场景：PC 接待屏、医生面诊前后说明方案
- 核心目标：让客户**看得懂、记得住、愿意问**
- 不是线上获客，不需要 SEO，不需要按官网逻辑设计

---

## 信息架构（5 个一级页面）

| Tab ID    | 页面标题 | 页面文件       |
|-----------|--------|----------------|
| `home`    | 首页    | Home.tsx       |
| `projects`| 服务项目 | Projects.tsx   |
| `experts` | 医生团队 | Experts.tsx    |
| `cases`   | 真实案例 | Cases.tsx      |
| `about`   | 诊所实力 | About.tsx      |

**命名原则**：
- 不用"关于我们"，用**诊所实力**（语义更直接）
- 不用"案例库"，用**真实案例**
- 不用"专家团"，用**医生团队**

---

## 核心设计原则

1. **先看懂，再看细** — 先给判断信息，再展开明细
2. **一屏一个重点** — 不在一屏内堆多个主题
3. **少文字，多图示** — 用标签、卡片、图标替代长段落
4. **大按钮，大卡片** — 适合平板触控，按钮高度 ≥ 44px
5. **讲解路径清晰** — 支持快速跳转，不依赖线性滚动

---

## 视觉规范

- 风格关键词：**简洁 / 专业 / 温馨**
- 避免：太挤、太乱、像表格、一屏塞满细项
- 优先：有分组、有重点、有留白、层级清晰
- 字号：一级标题 28–36px，正文 15–18px，标签 13–15px
- 主色：emerald-500（绿色系）

**避免的视觉错误**：
- 用陌生图库图带来廉价感或不真实感
- 把大量术语直接平铺成清单
- 在宽屏被明显横向拉伸（平板横屏必须稳定）

---

## 布局规范

- 平板竖屏：单列为主，双列卡片辅助
- 平板横屏 / PC：双列或三列，版心稳定，不因宽度变化拉伸
- 中等宽度优先两列或 1+1 分区，大屏再扩展
- `max-w-6xl` 作为页面版心

---

## 内容展示规范

### 价格
- 默认不在页面直接展示价格金额
- 例外：首页种植板块允许展示品牌与参考价格区间（帮助理解品牌定位和预算）

### 服务项目
- 一级：业务模块（6 个客户可懂的方向）
- 二级：sections（每个模块内的子分类）
- 总览页展示模块概览，详情页展开完整项目
- 命名用客户能懂的语言，不直接用专业术语做主标题

### 服务板块命名（6 个核心方向）
| 方向   | 显示名称          |
|--------|-----------------|
| 补牙根管 | 补牙 · 根管      |
| 拔牙外科 | 拔牙 · 外科      |
| 修复牙冠 | 修复 · 美学      |
| 牙周洁牙 | 牙周 · 洁牙      |
| 正畸矫正 | 矫正中心         |
| 种植    | 种植中心（单独处理）|

### 种植板块（重点内容，单独处理）
- 不与普通服务卡片混排
- 展示：品牌、系列、价格区间、牙冠类型、适合人群
- 结构：品牌对比卡 → 价格构成说明 → 额外费用 → 品牌特点速览
- 视觉：卡片式，避免大表格

## 技术规范

- **Tech Stack**: Vite + React + TypeScript + Tailwind CSS v4 + Motion
- **Icons**: lucide-react
- **数据**: `public/services.json`（运行时 fetch，`no-store` 防缓存）
- **部署**: GitHub Pages via GitHub Actions

### 开发命令
```bash
npm run dev          # 本地开发（固定端口 3001，不允许漂移到其他端口）
npm run dev:free     # 释放 3001 占用
npm run dev:reset    # 释放 + 重启
npm run build        # 打包（每次修改后必须执行）
npm run pages:sync   # build + commit + push（同步到 GitHub Pages）
```

### 完成一次修改后的标准流程
1. `npm run build` — 确认打包成功
2. `npm run pages:sync` — 同步到线上

## 讲解路径（预设 3 条）

**路径 1 · 项目导向**（客户有明确需求）
> 首页 → 服务项目 → 项目详情 → 价格说明 → 推荐医生 → 相关案例

**路径 2 · 信任建立**（客户还在犹豫）
> 首页 → 诊所优势 → 医生团队 → 诊所实力 → 真实案例 → 服务项目

**路径 3 · 案例带动**（客户想先看效果）
> 首页 → 真实案例 → 案例详情 → 主诊医生 → 对应项目 → 价格说明

---

## 开发约束

1. 每次修改涉及页面、数据、样式时，**必须同步执行 `npm run build`**
2. 每次完成后，**默认执行 `npm run pages:sync`** 保持线上同步
3. 信息量多时，**优先拆分层级**，不继续堆内容到同一屏
4. 修改 `public/sw.js` 时必须同步提升缓存版本
5. 不在首屏文案中使用内部产品语言（不说"这次改版"等）
